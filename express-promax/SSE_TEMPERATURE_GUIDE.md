# 温度数据实时监控 - SSE功能使用指南

## 🌡️ 功能概述

本项目实现了基于服务器发送事件(Server-Sent Events, SSE)的实时温度数据传输系统，支持：
- 实时温度数据采集和传输
- 多客户端并发连接
- 自动重连和错误恢复
- 数据可视化展示
- 性能优化和内存泄漏防护

## 🚀 快速开始

### 1. 启动服务器
```bash
npm start
```

### 2. 访问监控界面
打开浏览器访问：http://localhost:3000/temperature-monitor.html

### 3. 开始监控
- 点击"连接SSE"建立连接
- 点击"开始采集"启动温度数据采集
- 实时查看温度数据和趋势图

## 📡 API端点

### SSE端点
- **URL**: `/api/temperature/stream`
- **方法**: GET
- **描述**: 建立SSE连接，接收实时温度数据
- **数据格式**:
```
event: temperature
data: {"value": 25.5, "timestamp": "2026-03-07T01:45:00.000Z"}
```

### 控制API

#### 开始温度采集
- **URL**: `/api/temperature/control/start`
- **方法**: POST
- **参数**: `{ "interval": 1000 }` (采集间隔，毫秒)

#### 停止温度采集
- **URL**: `/api/temperature/control/stop`
- **方法**: POST

#### 获取采集状态
- **URL**: `/api/temperature/status`
- **方法**: GET

#### 设置温度范围
- **URL**: `/api/temperature/config/range`
- **方法**: POST
- **参数**: `{ "min": 15, "max": 35 }`

#### 重置温度
- **URL**: `/api/temperature/control/reset`
- **方法**: POST

#### 获取连接信息
- **URL**: `/api/temperature/connections`
- **方法**: GET

## 🛠️ 技术特性

### 性能优化
- **连接数限制**: 最大100个并发连接
- **内存管理**: 自动清理过期连接和数据
- **异步处理**: 非阻塞式数据传输
- **数据缓存**: 历史数据缓存和清理

### 错误处理
- **自动重连**: 客户端自动重连机制
- **连接超时**: 5分钟无活动连接自动清理
- **异常处理**: 服务端异常捕获和处理
- **数据验证**: 输入参数验证和错误返回

### 数据格式
- **标准SSE格式**: 符合W3C SSE规范
- **JSON数据**: 结构化温度数据
- **时间戳**: ISO 8601标准时间格式
- **心跳机制**: 30秒心跳保持连接

## 🔧 配置选项

### 采集参数
- **采集间隔**: 500ms - 10000ms (默认2000ms)
- **温度范围**: -50°C - 100°C (默认15°C - 35°C)
- **数据点数量**: 最大50个数据点显示

### 连接参数
- **最大连接数**: 100个并发连接
- **连接超时**: 5分钟无活动超时
- **心跳间隔**: 30秒发送心跳包
- **重连机制**: 指数退避重连策略

## 🐛 故障排除

### 常见问题

1. **连接失败**
   - 检查服务器是否启动
   - 确认端口号正确
   - 查看防火墙设置

2. **数据不更新**
   - 确认温度采集已启动
   - 检查SSE连接状态
   - 查看浏览器控制台

3. **内存占用高**
   - 检查连接数是否过多
   - 确认清理机制正常
   - 重启服务释放资源
