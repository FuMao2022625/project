# 数据库结构变更文档

## 1. 变更概述

本次数据库结构优化涉及以下几个方面的变更：

- **表结构优化**：修改字段数据类型
- **索引优化**：添加复合索引，移除不必要的索引
- **连接池配置优化**：调整连接池参数
- **监控功能添加**：新增数据库性能监控和日志功能

## 2. 详细变更内容

### 2.1 表结构变更

#### 2.1.1 sensor_data表

| 字段名 | 变更前 | 变更后 | 变更原因 | 影响 |
|-------|-------|-------|---------|------|
| battery | INT NOT NULL | TINYINT UNSIGNED NOT NULL | 电池电量取值范围为0-100，使用TINYINT更合适 | 减少存储空间，提高查询性能 |

**SQL变更语句**：
```sql
ALTER TABLE sensor_data MODIFY COLUMN battery TINYINT UNSIGNED NOT NULL COMMENT '电池电量';
```

### 2.2 索引变更

#### 2.2.1 sensor_data表

| 索引名 | 变更类型 | 变更内容 | 变更原因 | 影响 |
|-------|---------|---------|---------|------|
| idx_temperature | 移除 | 单字段索引 | 使用率低，占用存储空间 | 减少索引维护开销，提高写入性能 |
| idx_humidity | 移除 | 单字段索引 | 使用率低，占用存储空间 | 减少索引维护开销，提高写入性能 |
| idx_smoke_level | 移除 | 单字段索引 | 使用率低，占用存储空间 | 减少索引维护开销，提高写入性能 |
| idx_human_detected | 移除 | 单字段索引 | 使用率低，占用存储空间 | 减少索引维护开销，提高写入性能 |
| idx_device_timestamp | 添加 | 复合索引(device_id, timestamp) | 优化按设备和时间范围的查询 | 提高查询性能，特别是时间范围查询 |

**SQL变更语句**：
```sql
-- 添加复合索引
CREATE INDEX idx_device_timestamp ON sensor_data(device_id, timestamp);

-- 移除不必要的索引
DROP INDEX idx_temperature ON sensor_data;
DROP INDEX idx_humidity ON sensor_data;
DROP INDEX idx_smoke_level ON sensor_data;
DROP INDEX idx_human_detected ON sensor_data;
```

#### 2.2.2 device_data表

| 索引名 | 变更类型 | 变更内容 | 变更原因 | 影响 |
|-------|---------|---------|---------|------|
| idx_device_type_timestamp | 添加 | 复合索引(device_id, data_type, timestamp) | 优化按设备、数据类型和时间范围的查询 | 提高查询性能，特别是多条件组合查询 |

**SQL变更语句**：
```sql
-- 添加复合索引
CREATE INDEX idx_device_type_timestamp ON device_data(device_id, data_type, timestamp);
```

### 2.3 连接池配置变更

| 配置项 | 变更前 | 变更后 | 变更原因 | 影响 |
|-------|-------|-------|---------|------|
| connectionLimit | 20 | 15 | 更适合当前系统规模，减少资源占用 | 提高连接管理效率，减少内存使用 |
| acquireTimeout | 10000 | 移除 | 无效参数，导致配置警告 | 消除配置警告，提高连接池稳定性 |

**配置变更**：
```javascript
// 变更前
const pool = mysql.createPool({
  ...DB_CONFIG,
  waitForConnections: true,
  connectionLimit: 20,
  queueLimit: 0,
  enableKeepAlive: true,
  keepAliveInitialDelay: 30000,
  connectTimeout: 10000,
  acquireTimeout: 10000
});

// 变更后
const pool = mysql.createPool({
  ...DB_CONFIG,
  waitForConnections: true,
  connectionLimit: 15, // 优化连接池大小
  queueLimit: 0,
  enableKeepAlive: true,
  keepAliveInitialDelay: 30000,
  connectTimeout: 10000
});
```

### 2.4 监控功能添加

#### 2.4.1 DatabaseMonitor类

**功能**：监控数据库查询执行时间和连接池状态

**主要方法**：
- `recordQuery(query, duration)`：记录查询执行时间
- `getPoolStatus()`：获取连接池状态

#### 2.4.2 queryWithMonitoring方法

**功能**：增强的查询方法，添加性能监控

**参数**：
- `sql`：SQL查询语句
- `params`：查询参数

**返回值**：查询结果

**特点**：
- 记录查询执行时间
- 捕获并记录错误信息
- 提供详细的错误日志

#### 2.4.3 定期检查任务

**功能**：定期检查连接池状态

**配置**：每60秒检查一次连接池状态

**SQL变更语句**：无（纯代码实现）
